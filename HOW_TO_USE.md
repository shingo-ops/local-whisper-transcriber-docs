## リアルタイム録音

### 使い方

**1. メニューで `2` を選択**

```
選択してください (0-7): 2
```

**2. マイクを選択**

```
デフォルトのマイクを使用しますか? (y/n) [y]:
```

初めての場合は `y` を入力（または単にEnter）

**3. 録音方法を選択**

```
録音方法を選択:
1. 自分で停止するまで録音（推奨）
2. 指定時間だけ録音

選択 (1/2) [1]:
```

会議の場合は `1` を選択（または単にEnter）

**4. 録音開始**

```
準備ができたら Enter キーを押してください...
```

Enterを押すと録音開始！

```
🔴 録音中... (Ctrl+C で停止)
```

**5. 録音停止**

会議が終わったら **Ctrl+C** を押す

**6. 自動処理**

録音停止後、自動的に：
- ✅ 文字起こし（約10分/30分の会議）
- ✅ 議事録生成（約30秒）

**7. 結果確認**

```
完了！

結果は以下に保存されました:
- 録音: audio_files/recording_20260311_150000.wav
- 文字起こし: output/recording_20260311_150000_transcript.txt
- 議事録: output/recording_20260311_150000_minutes.md
```

## 既存ファイルの処理

### 音声ファイルを文字起こし + 議事録生成

**1. メニューで `1` を選択**

```
選択してください (0-7): 1
```

**2. ファイルパスを入力**

```
音声ファイルのパスを入力:
```

**3つの入力方法:**

#### 方法A: 絶対パスを入力
```
音声ファイルのパスを入力: /Users/yourname/Downloads/meeting.mp3
```

#### 方法B: チルダ（~）を使う
```
音声ファイルのパスを入力: ~/Downloads/meeting.mp3
```

#### 方法C: ドラッグ&ドロップ
1. `音声ファイルのパスを入力: ` と表示されたら
2. Finderから音声ファイルをターミナルにドラッグ&ドロップ
3. パスが自動入力される
4. Enterを押す

**3. 言語を選択**

```
言語コード (ja/en/auto) [ja]:
```

- 日本語の場合: `ja` または Enter
- 英語の場合: `en`
- 自動検出: `auto`

**4. 処理開始**

自動的に文字起こしと議事録生成が実行されます。

**5. 結果確認**

```
output/
├── meeting_transcript.txt  # 文字起こし結果
└── meeting_minutes.md      # 議事録
```

## Google Sheets連携

既存のOpenAI Whisperアプリと連携できます。

### 設定方法

#### 1. Google Cloud Consoleで認証情報を取得

1. [Google Cloud Console](https://console.cloud.google.com/)にアクセス
2. プロジェクトを作成
3. Google Sheets APIを有効化
4. OAuth 2.0認証情報を作成（デスクトップアプリ）
5. `credentials.json` をダウンロード

#### 2. credentials.json を配置

```bash
# プロジェクトルートに配置
mv ~/Downloads/credentials.json ~/sales-ops-with-claude/02_apps/whisper-tts-tool/local-whisper-ollama/
```

#### 3. .env ファイルを編集

```bash
nano .env
```

以下を追加：
```bash
GOOGLE_CREDENTIALS_PATH=./credentials.json
SPREADSHEET_ID=your_spreadsheet_id_here
SHEET_NAME=音声解析ログ
```

**スプレッドシートIDの取得方法:**

スプレッドシートのURLから：
```
https://docs.google.com/spreadsheets/d/[このIDをコピー]/edit
```

### 使い方

**1. メニューで `6` を選択**

```
選択してください (0-7): 6
```

**2. 初回のみ認証**

ブラウザが開くので、Googleアカウントでログイン

**3. 自動処理**

- スプレッドシートから未処理の文字起こし結果を取得
- ローカルLLMで議事録生成
- 結果をスプレッドシートに追記

## よくある質問

### Q1: セットアップは毎回必要？

**A: いいえ、初回のみです。**

2回目以降は `./run.sh` だけで起動できます。

---

### Q2: ターミナルを閉じたら設定は消える？

**A: いいえ、消えません。**

セットアップした内容（モデル、仮想環境等）は永続的に保存されます。

ただし、ターミナルを新しく開くたびに `./run.sh` は必要です。

---

### Q3: 処理時間はどれくらい？

| 音声長 | 文字起こし | 議事録 | 合計 |
|--------|----------|--------|------|
| 10分 | 約3分 | 約10秒 | 約3分10秒 |
| 30分 | 約10分 | 約30秒 | 約10分30秒 |
| 1時間 | 約20分 | 約1分 | 約21分 |

---

### Q4: 対応している音声ファイル形式は？

以下の形式に対応：
- ✅ mp3
- ✅ m4a
- ✅ wav
- ✅ webm
- ✅ mp4（音声のみ）
- ✅ その他（FFmpegが対応する形式）

---

### Q5: コストは？

**完全無料です！**

すべてローカルで処理するため、APIコストは0円です。

比較：
- OpenAI Whisper API: 約27円/30分
- OpenAI GPT-4: 約50円/議事録
- **本ツール: 0円**

---

### Q6: オフラインでも使える？

**はい、使えます！**

初回のモデルダウンロードが完了すれば、完全オフラインで動作します。

---

### Q7: 精度はどれくらい？

| 項目 | OpenAI | 本ツール |
|------|--------|---------|
| 文字起こし精度 | ★★★★★ | ★★★★☆ |

## パフォーマンス最適化

### 処理を高速化したい

`.env` ファイルを編集：

```bash
# より小さいモデルを使用
WHISPER_MODEL=medium      # デフォルト: large-v3
OLLAMA_MODEL=qwen2.5:3b   # デフォルト: qwen2.5:7b
```

変更後、次回起動時から反映されます。

---

### メモリ不足の場合

```bash
# さらに軽量な設定
WHISPER_MODEL=small
WHISPER_COMPUTE_TYPE=int8
OLLAMA_MODEL=gemma2:2b
```
